OpenSSL Heartbleed Bug

It has recently been discovered that Olanto products may be vulnerable to the OpenSSL Heartbleed bug. 

The Heartbleed vulnerability and Olanto products

The Olanto systems are fine:

  • The olanto.org website
  • The systems and demos (myCAT and myMT).


So customers that have not deployed the HTTPS mode for myCAT and myMT are not affected by the vulnerability.

How to test the vulnerability

Use this utility: http://filippo.io/Heartbleed

Deployed products :

  • Ubuntu 10.04 LTS (no problem)
  • Ubuntu 12.04 LTS (update if you use openSSL)
  • Tomcat 7.0.29 (change mode if HTTPS is used in ARP mode, waiting for a new Tomcat distribution).


What to do with Ubuntu 12.04 LTS

To update Ubuntu:
> apt-get dist-upgrade

To check out the release number:
> openssl version -a

The result must show a date equal to or later than the 7th of April.

Example:
OpenSSL 1.0.1 March 14, 2012
built on: Mon Apr 7 2014 8:33:29 p.m. UTC
...

What to do with Tomcat 7.0.29

If the system uses ARP then you must use the other method: JSSE . Follow this documentation:
http://tomcat.apache.org/tomcat-3.3-doc/tomcat-ssl-howto.html

Notes:
If you are using OpenSSL, you were and still are vulnerable. It is strongly recommended to renew your SSL certificates.

Le bug OpenSSL Heartbleed

Il a été récemment découvert que les produits Olanto peuvent être vulnérables au bug OpenSSL Heartbleed. 

La Vulnérabilité Heartbleed et les produits Olanto

Les systèmes Olanto sont sécures:

  • Le site Web olanto.org
  • Les systèmes et démos (myCAT et myMT).

Les clients qui n'ont pas déployé le mode HTTPS pour myCAT et myMT ne sont pas affectés par la vulnérabilité

Comment tester la vulnérabilité
Utiliser cet outil: http://filippo.io/Heartbleed

Produits déjà déployés :

  • Ubuntu 10.04 LTS (pas de problèmes)
  • Ubuntu 12.04 LTS (mettre à jour si vous utilisez OpenSSL)
  • Tomcat 7.0.29 (changer le mode si HTTPS est utilisé en mode ARP, attente d'une nouvelle distribution de Tomcat)

Comment prodéder avec Ubuntu 12.04 LTS

Pour mettre à jour Ubuntu:
> apt-get dist-upgrade

Pour vérifier le numéro de la distribution:
> openssl version -a

Le résultat doit montrer une date égale ou plus tard que le 7 Avril 2014.

Exemple:
OpenSSL 1.0.1 March 14, 2012
built on: Mon Apr 7 2014 8:33:29 p.m. UTC
...

Comment procéder avec Tomcat 7.0.29

Si le système utilise ARP alors vous devez utiliser l'autre méthode: JSSE. Veuillez suivre cette documentation:
http://tomcat.apache.org/tomcat-3.3-doc/tomcat-ssl-howto.html

Notes:
Si vous utilisez OpenSSL, vous étiez et êtes toujours vulnérables. Il est fortement recommandé de renouveler vos certificats SSL.
